# Builder's Journal 001 — Finding the Real Problem

## Initial Assumption

At first, the project seemed to be about building a System Analyst Agent.

Possible features included:

- summarizing meeting notes
- generating tickets
- extracting tasks
- supporting daily SA work

The starting question was:

> What can AI help me do?

## Discovery

After discussing real workplace examples, the problem became clearer.

The issue was not that there was not enough documentation.

The issue was that information was not flowing correctly.

Documents existed, but they were too long, too undifferentiated, and not designed for the people who needed to use them.

## Key Shift

From:

> Build an SA Agent.

To:

> Understand how knowledge should flow in an engineering team.

## Product Insight

The project should not start from AI capabilities.

It should start from human workflow.
