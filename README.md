# Sherry Lab

A public research notebook documenting my journey from Backend Engineer to System Analyst, and how that journey gradually led me into AI Product Building.

This repository is not a software project.

It is a thinking project.

I started documenting my work because I realized I was not only learning AI. I was learning how knowledge flows inside engineering teams, how decisions are made, and how context should be delivered to different people.

Eventually, I realized that these ideas aligned surprisingly well with modern AI Agent architectures.

This repository captures that discovery.

## Core Themes

- Knowledge Routing
- Workflow Design
- Product Thinking
- Cognitive Load
- Context Engineering
- Decision History
- Human-AI Collaboration
- Workflow Abstraction

## Core Principles

> Knowledge should be routed, not broadcast.

> Context is a resource. Spend it wisely.

> Good documentation preserves decisions.

> Different roles require different context.
