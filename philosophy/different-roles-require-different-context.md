# Different Roles Require Different Context

## Principle

> Different roles require different context.

## Examples

Engineers need:

- goal
- scope
- input
- output
- acceptance criteria

PMs need:

- progress
- blockers
- milestones
- risks

QAs need:

- expected behavior
- edge cases
- acceptance criteria

Stakeholders need:

- outcome
- timeline
- impact

## Product Implication

The same source material should be distilled into different role-specific outputs.
